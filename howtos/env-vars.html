Environment Variables and JVM options 🔗

How to pass environment variables to run/test 🔗

You need to use the forkEnv property in module config:

forkEnv {
  ["MY_VAR"] = "true"
}

Then you can read this env var when running your app with deder exec -t run, or when you run tests with deder exec -t test.

Note that fork must be set to true on test module in order for this to work!

How to pass JVM options to run/test 🔗

You need to use the jvmOptions property in module config:

jvmOptions {
  "-Xmx128m"
  "-XX:+PrintCompilation"
}
